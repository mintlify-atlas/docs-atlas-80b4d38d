## Overview

The CO2 Monitor uses Chart.js and react-chartjs-2 to create dynamic, responsive line charts that visualize CO2 levels over time. The visualization automatically updates every 5 seconds as new data arrives.

## Chart Library Integration

### Dependencies

The project uses two key packages for charting:

```json package.json
{
  "chart.js": "^4.4.6",
  "react-chartjs-2": "^5.2.0"

```

### Chart.js Registration

Before using Chart.js components, required modules must be registered:

```jsx src/components/CO2Dashboard.js
import { Line } from "react-chartjs-2";
import { 
  Chart as ChartJS, 
  CategoryScale, 
  LinearScale, 
  PointElement, 
  LineElement, 
  Title, 
  Tooltip, 
  Legend 
} from 'chart.js';

ChartJS.register(
  CategoryScale, 
  LinearScale, 
  PointElement, 
  LineElement, 
  Title, 
  Tooltip, 
  Legend
);
```

<Info>
  Chart.js v4 uses a modular architecture. Only the components you register are included in your bundle, reducing the final build size.
</Info>

### Module Purposes

| Module | Purpose |
|--------|--------|
| `CategoryScale` | X-axis scale for categorical data (timestamps) |
| `LinearScale` | Y-axis scale for numerical data (CO2 values) |
| `PointElement` | Renders individual data points on the chart |
| `LineElement` | Draws lines connecting data points |
| `Title` | Adds titles to chart and axes |
| `Tooltip` | Interactive tooltips on hover |
| `Legend` | Dataset labels and color indicators |

## Chart Data Structure

The chart receives data in Chart.js's expected format:

```jsx src/components/CO2Dashboard.js
const data = {
    labels: timestamps,
    datasets: [
        {
            label: "Niveles de CO2 (ppm)",
            data: co2Data,
            fill: false,
            backgroundColor: "rgba(75,192,192,1)",
            borderColor: "rgba(75,192,192,0.2)",
        },
    ],
};
```

### Data Object Breakdown

<Steps>
  <Step title="Labels Array">
    The `labels` array contains formatted timestamps for the x-axis (e.g., ["10:30:00", "10:30:05", "10:30:10"])
  </Step>
  
  <Step title="Datasets Array">
    Contains one or more dataset objects. This app uses a single dataset for CO2 readings
  </Step>
  
  <Step title="Dataset Properties">
    Each dataset defines:
    - `label`: Legend text
    - `data`: Array of CO2 values
    - `fill`: Whether to fill area under the line
    - Color properties for styling
  </Step>
</Steps>

<Note>
  The `timestamps` and `co2Data` arrays are React state variables that update every 5 seconds, automatically triggering a chart re-render.
</Note>

## Chart Configuration

The Line chart component is configured with comprehensive options:

```jsx src/components/CO2Dashboard.js
<Line data={data} options={{
    responsive: true,
    maintainAspectRatio: false,
    scales: {
        x: { 
            ticks: { color: "#555" }, 
            title: { display: true, text: 'Tiempo' } 
        },
        y: { 
            ticks: { color: "#555" }, 
            title: { display: true, text: 'CO2 (ppm)' } 
        },
    },
    plugins: {
        tooltip: {
            callbacks: {
                label: (context) => `${context.parsed.y} ppm`
            }
        }
    }
}} />
```

## Time Formatting for Labels

The `formatTime` function transforms API timestamps into clean chart labels:

```jsx src/components/CO2Dashboard.js
const formatTime = (timestamp) => {
    const [datePart, timePartWithPeriod] = timestamp.split(", ");
    const [day, month, year] = datePart.split("/");
    const [timePart, period] = timePartWithPeriod.split(" ");
    let [hours, minutes] = timePart.split(":");
    if (period.toLowerCase() === "p.m." && hours !== "12") {
        hours = parseInt(hours) + 12;
    } else if (period.toLowerCase() === "a.m." && hours === "12") {
        hours = "00";
    }
    const isoDate = `20${year}-${month}-${day}T${hours}:${minutes}:00`;
    const date = new Date(isoDate);
    if (isNaN(date)) {
        console.error("Fecha no válida:", timestamp);
        return "Invalid Date";
    }
    return date.toLocaleTimeString('es-MX', {
        hour: '2-digit',
        minute: '2-digit',
        second: '2-digit',
        hour12: false
    });
};
```

### Transformation Process

<Steps>
  <Step title="Parse Input">
    Input: `"13/03/26, 10:30 a.m."`
    Splits into date and time components
  </Step>
  
  <Step title="Convert to 24-Hour">
    Handles a.m./p.m. conversion, including edge cases like 12:00 a.m. and 12:00 p.m.
  </Step>
  
  <Step title="Create Date Object">
    Builds ISO 8601 string and creates JavaScript Date object
  </Step>
  
  <Step title="Format Output">
    Output: `"10:30:00"`
    Uses Mexican Spanish locale in 24-hour format
  </Step>
</Steps>

<Warning>
  Error handling prevents chart crashes from malformed timestamps by returning "Invalid Date" and logging errors to the console.
</Warning>

<Tip>
  Using RGBA colors with alpha transparency creates a softer visual appearance while maintaining point visibility.
</Tip>

### Line Style

```jsx
fill: false,
```

Setting `fill: false` creates a line chart without area fill, focusing attention on the trend line and data points.

## Dynamic Updates

The chart automatically re-renders when state changes:

1. New data arrives every 5 seconds
2. `co2Data` and `timestamps` state arrays update
3. React detects state change
4. Chart component re-renders with new data
5. Chart.js smoothly animates the update

<Info>
  Chart.js includes built-in animations, so new points smoothly appear and old points slide left as the data window moves.
</Info>

## Customization Options

You can customize the chart by modifying:

- **Window size**: Change `if (newData.length > 10)` to show more/fewer points
- **Colors**: Update `backgroundColor` and `borderColor` in the dataset
- **Axis labels**: Modify the `title.text` values in scale configuration
- **Tooltip format**: Customize the `callbacks.label` function
- **Chart type**: Replace `Line` with `Bar`, `Scatter`, or other Chart.js components
