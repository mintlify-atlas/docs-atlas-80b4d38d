## Overview

The CO2 Monitor is built as a Progressive Web App (PWA), enabling it to be installed on mobile devices and desktops like a native application. This provides users with a more app-like experience, including home screen icons, standalone display mode, and improved performance.

## What is a PWA?

Progressive Web Apps are web applications that use modern web capabilities to deliver an app-like experience to users. They combine the best of web and native apps.

## Web App Manifest

The PWA functionality is enabled through the `manifest.json` file, which defines how the app appears when installed:

```json public/manifest.json
{
  "short_name": "CO2 Monitor",
  "name": "Sistema de Monitoreo de CO2",
  "icons": [
    {
      "src": "favicon.ico",
      "sizes": "64x64 32x32 24x24 16x16",
      "type": "image/x-icon"
    },
    {
      "src": "logo192.png",
      "type": "image/png",
      "sizes": "192x192"
    },
    {
      "src": "logo512.png",
      "type": "image/png",
      "sizes": "512x512"
    }
  ],
  "start_url": ".",
  "display": "standalone",
  "theme_color": "#000000",
  "background_color": "#ffffff",
  "version": "1.0.0"
}
```

### Icon Sizes

The manifest defines three icon variants:

| Icon | Size | Usage |
|------|------|-------|
| `favicon.ico` | 16x16 to 64x64 | Browser tabs and bookmarks |
| `logo192.png` | 192x192 | Android home screen, app launcher |
| `logo512.png` | 512x512 | Splash screens, high-DPI displays |

<Info>
  Providing multiple icon sizes ensures optimal display quality across different devices and contexts, from small notification icons to large splash screens.
</Info>

## HTML Integration

The manifest is linked in the HTML document:

```html public/index.html
<head>
    <meta charset="utf-8" />
    <link rel="icon" href="%PUBLIC_URL%/favicon.ico" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <meta name="theme-color" content="#000000" />
    <meta name="description"
      content="Sistema de monitoreo de CO2 para invernaderos. Monitorea los niveles de CO2 en tiempo real para un ambiente saludable y controlado. Mantén la calidad del aire en óptimas condiciones para tus cultivos." />
    <link rel="apple-touch-icon" href="%PUBLIC_URL%/logo192.png" />
    <link rel="manifest" href="%PUBLIC_URL%/manifest.json" />
    <title>CO2</title>
</head>
```

### Key Meta Tags

#### Viewport Configuration
```html
<meta name="viewport" content="width=device-width, initial-scale=1" />
```
Ensures proper responsive behavior on mobile devices by setting the viewport width to the device width.

#### Theme Color
```html
<meta name="theme-color" content="#000000" />
```
Sets the color of browser UI elements (like the status bar on Android) to black, matching the app's design.

#### App Description
```html
<meta name="description" content="Sistema de monitoreo de CO2 para invernaderos..." />
```
Provides a detailed description for search engines and when sharing the app link.

<Note>
  The description emphasizes the app's purpose for greenhouse monitoring and maintaining optimal air quality for crops.
</Note>

#### Apple Touch Icon
```html
<link rel="apple-touch-icon" href="%PUBLIC_URL%/logo192.png" />
```
Specifies the icon used when installing the app on iOS devices.

## Installation Process

### On Mobile Devices (Android/iOS)

<Steps>
  <Step title="Open in Browser">
    Navigate to the app URL in Chrome (Android) or Safari (iOS)
  </Step>
  
  <Step title="Add to Home Screen">

  </Step>

### On Desktop (Chrome, Edge)

<Steps>
  <Step title="Open in Browser">
    Visit the app URL in Chrome or Edge
  </Step>
  
  <Step title="Install Prompt">
    Click the install icon in the address bar or use browser menu: "Install CO2 Monitor"

  <Step title="Confirmation">

  <Step title="Launch">

<Tip>
  Desktop PWA installations create shortcuts that can be pinned to the taskbar or dock, making the app as accessible as any native application.
</Tip>

## Start URL Configuration

The manifest specifies where the app should start:

```json
"start_url": "."
```

The `"."` value means the app starts at the manifest's location (the root), ensuring users always begin at the main dashboard when launching from the home screen icon.

## Service Worker Support

The project is built with Create React App, which includes built-in service worker support through `react-scripts`. While not explicitly registered in the current `src/index.js`, the infrastructure is available:

```jsx src/index.js
import React from 'react';
import ReactDOM from 'react-dom/client';
import './index.css';
import App from './App';
import reportWebVitals from './reportWebVitals';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);

reportWebVitals();
```

### Enabling Service Worker

To enable offline capabilities, you can activate the service worker by modifying `src/index.js`:

```jsx
import * as serviceWorkerRegistration from './serviceWorkerRegistration';

// After ReactDOM.render()...
serviceWorkerRegistration.register();
```

<Note>
  Create React App's service worker implementation provides:
  - Offline-first caching strategy
  - Automatic asset caching
  - Cache-first for app shell
  - Network-first for API calls
</Note>
